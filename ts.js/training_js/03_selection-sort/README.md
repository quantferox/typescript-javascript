# 🔍 Сортировка выбором (Selection Sort)

## Аналогия из жизни

Ты перебираешь стопку книг и хочешь разложить их по толщине.  
Смотришь на **все** книги → берёшь **самую тонкую** → кладёшь в начало новой стопки.  
Снова смотришь на **оставшиеся** → берёшь **самую тонкую** → кладёшь следующей.  
И так до конца.

Ключевое отличие от пузырька и вставок: ты ищешь **лучший вариант среди всех оставшихся**, а не просто сравниваешь соседей.

---

## Как работает алгоритм

```
[5, 3, 8, 4, 2]

Шаг 1: Ищем min в [5,3,8,4,2] → это 2 (индекс 4)
  Меняем arr[0] ↔ arr[4]: [2, 3, 8, 4, 5] ✅

Шаг 2: Ищем min в [3,8,4,5] → это 3 (индекс 1, уже на месте)
  Обмен не нужен: [2, 3, 8, 4, 5] ✅

Шаг 3: Ищем min в [8,4,5] → это 4 (индекс 3)
  Меняем arr[2] ↔ arr[3]: [2, 3, 4, 8, 5] ✅

Шаг 4: Ищем min в [8,5] → это 5 (индекс 4)
  Меняем arr[3] ↔ arr[4]: [2, 3, 4, 5, 8] ✅

Итого: 3 обмена для массива из 5 элементов
```

## Код с объяснением

```javascript
function selectionSort(arr) {
  const n = arr.length;

  for (let i = 0; i < n - 1; i++) {
    let minIndex = i;               // предполагаем, что i-й — минимум

    for (let j = i + 1; j < n; j++) {
      if (arr[j] < arr[minIndex]) {
        minIndex = j;               // нашли новый минимум
      }
    }

    if (minIndex !== i) {
      [arr[i], arr[minIndex]] = [arr[minIndex], arr[i]];  // ОДИН обмен!
    }
  }

  return arr;
}
```

## Сложность

| Случай     | Время  | Почему?                                          |
|------------|--------|--------------------------------------------------|
| Лучший     | O(n²)  | Даже для отсортированного нужно просматривать всё |
| Средний    | O(n²)  | Всегда n*(n-1)/2 сравнений                       |
| Худший     | O(n²)  | То же самое                                      |
| **Память** | **O(1)** | In-place                                       |

**Важно:** в отличие от Bubble и Insertion Sort, входные данные не влияют на время. Отсортированный и обратный массив занимают **одинаковое** время.

---

## Уникальное свойство: минимальное число записей

Selection Sort делает **максимум n−1 обменов**, независимо от данных.

Для сравнения:
- Bubble Sort: до n²/2 обменов
- Insertion Sort: до n²/2 сдвигов
- Selection Sort: до n−1 обменов

Это критично, если **запись в память дорогостоящая** (например, флэш-память, EEPROM, или структуры данных с дорогим присваиванием).

---

## Достоинства и недостатки

| ✅ Плюсы | ❌ Минусы |
|----------|-----------|
| Минимальное количество обменов | Всегда O(n²), нет адаптации |
| In-place | Нестабильный |
| Простой для понимания | Медленнее Insertion Sort на практике |
