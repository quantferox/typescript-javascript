# 🔀 Сортировка слиянием (Merge Sort)

## Аналогия из жизни

У тебя две стопки карточек с числами — каждая уже **отсортирована по возрастанию**.  
Тебе нужно объединить их в одну отсортированную стопку.

Это просто: смотришь на верхние карточки обеих стопок, берёшь меньшую, кладёшь в результат. Повторяешь до конца.

Merge Sort именно это и делает — разбивает массив до отдельных элементов (они "отсортированы" автоматически), а потом **сливает** обратно.

---

## Как работает алгоритм

```
Разделение (сверху вниз):
[38, 27, 43, 3, 9, 82, 10]
        ↓ делим пополам
[38, 27, 43]    [3, 9, 82, 10]
      ↓                ↓
[38, 27] [43]    [3,9] [82,10]
   ↓                ↓
[38][27] [43]  [3][9] [82][10]   ← базовый случай!

Слияние (снизу вверх):
[38][27] → сливаем → [27, 38]
[27,38] + [43] → [27, 38, 43]
[3][9]  → [3, 9]
[82][10] → [10, 82]
[3,9] + [10,82] → [3, 9, 10, 82]
[27,38,43] + [3,9,10,82] → [3, 9, 10, 27, 38, 43, 82] ✅
```

---

## Функция merge — главная идея

```javascript
function merge(left, right) {
  const result = [];
  let i = 0, j = 0;

  while (i < left.length && j < right.length) {
    if (left[i] <= right[j]) {   // берём меньший из двух
      result.push(left[i++]);
    } else {
      result.push(right[j++]);
    }
  }

  // Добавляем то, что осталось в одной из частей
  return result.concat(left.slice(i)).concat(right.slice(j));
}
```

## Сложность

| Случай     | Время      | Почему?                                          |
|------------|------------|--------------------------------------------------|
| Лучший     | O(n log n) | Дерево рекурсии глубиной log n, на каждом уровне O(n) |
| Средний    | O(n log n) | То же самое                                      |
| Худший     | O(n log n) | **Всегда** — главное преимущество над Quicksort  |
| **Память** | **O(n)**   | Нужен дополнительный массив для слияния          |

## Когда применять?

- Нужна **гарантированная** O(n log n) (в отличие от Quicksort)
- Нужна **стабильность** (порядок одинаковых элементов сохраняется)
- Сортировка **связных списков** (Merge Sort отлично работает, Quicksort — нет)
- **Внешняя сортировка** (файлы не помещаются в память) — Merge Sort легко адаптируется
- `Array.prototype.sort` в Firefox и Safari использует Merge Sort
