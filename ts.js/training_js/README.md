|--------------------|:-------------:|:--------------:|:-------------:|:------:|:-----------:|

## 🧠 Когда какой алгоритм использовать?

- **Пузырьком** — только для учёбы. В реальном коде не используют.
- **Вставками** — идеален для **почти отсортированных** или **маленьких** массивов (< 20 элементов).
- **Выбором** — когда **запись в память дорогая** (минимальное число обменов).
- **Быстрая сортировка** — золотой стандарт. Используется в большинстве стандартных библиотек (`Array.prototype.sort` в V8).
- **Слиянием** — когда нужна **стабильность** и предсказуемый O(n log n). Хороша для связных списков и внешней сортировки.
- **Кучей** — когда нужен **гарантированный O(n log n)** без дополнительной памяти.

## 💡 Важные понятия

**Стабильная сортировка** — сохраняет относительный порядок одинаковых элементов.  
Пример: если в массиве `[{id:1, age:25}, {id:2, age:25}]` оба элемента с одинаковым `age`, стабильная сортировка гарантирует, что `id:1` окажется раньше `id:2`.

**In-place сортировка** — не требует дополнительной памяти, сортирует прямо в исходном массиве.

**O-нотация** — способ описать, как растёт время работы алгоритма при увеличении размера данных:
- O(n) — линейный рост (отлично)
- O(n log n) — почти линейный (хорошо)
- O(n²) — квадратичный рост (плохо для больших данных)
