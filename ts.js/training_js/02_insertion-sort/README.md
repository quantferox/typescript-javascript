# 🃏 Сортировка вставками (Insertion Sort)

## Аналогия из жизни

Представь, что ты берёшь карты из колоды одну за другой и вставляешь каждую на нужное место в руке.  
Левая часть — уже **отсортированные** карты в руке.  
Правая часть — ещё не взятые карты из колоды.

Каждый шаг: берёшь следующую карту и находишь для неё правильное место слева.

---

## Как работает алгоритм

```
[5, 3, 8, 4, 2]

Шаг 1: key = 3
  [5 > 3] → сдвигаем 5 вправо → [_, 5, 8, 4, 2]
  Вставляем 3 → [3, 5, 8, 4, 2] ✅

Шаг 2: key = 8
  [5 < 8] → стоп → вставляем на месте
  [3, 5, 8, 4, 2] ✅ (не изменился)

Шаг 3: key = 4
  [8 > 4] → сдвигаем 8 → [3, 5, _, 8, 2]
  [5 > 4] → сдвигаем 5 → [3, _, 5, 8, 2]
  [3 < 4] → стоп → вставляем 4 → [3, 4, 5, 8, 2] ✅

Шаг 4: key = 2
  Сдвигаем 8, 5, 4, 3 → [_, 3, 4, 5, 8]
  Вставляем 2 → [2, 3, 4, 5, 8] ✅
```

## Код с объяснением

```javascript
function insertionSort(arr) {
  for (let i = 1; i < arr.length; i++) {
    const key = arr[i];  // "вытаскиваем" элемент
    let j = i - 1;

    // Сдвигаем элементы правее, пока они больше key
    while (j >= 0 && arr[j] > key) {
      arr[j + 1] = arr[j];  // сдвигаем вправо
      j--;
    }

    arr[j + 1] = key;  // вставляем на нужное место
  }
  return arr;
}
```

Ключевая деталь: мы не **меняем** элементы местами (как пузырёк), а **сдвигаем** их — это быстрее.

## Сложность

| Случай        | Время  | Почему?                                        |
|---------------|--------|------------------------------------------------|
| Лучший (✅)   | O(n)   | Уже отсортирован → на каждом шаге 1 сравнение |
| Средний       | O(n²)  | В среднем сдвигаем половину отсортированной части |
| Худший (❌)   | O(n²)  | Обратный порядок → сдвигаем всю часть каждый раз |
| **Память**    | **O(1)**| Только переменная `key`                        |

---

## Почему Insertion Sort лучше Bubble Sort?

- Сдвиги элементов быстрее, чем попарные обмены (меньше операций записи)
- Одна переменная `key` вместо временной переменной для обмена
- В V8 (Node.js/Chrome) для маленьких массивов внутри `Array.sort` используется именно Insertion Sort

## Достоинства и недостатки

| ✅ Плюсы | ❌ Минусы |
|----------|-----------|
| Отличен для малых и почти готовых массивов | Медленно на больших случайных данных |
| Стабильный | O(n²) в худшем случае |
| In-place | |
| Адаптивный (быстрее на почти отсортированных) | |
| Онлайн-алгоритм (можно добавлять элементы "на лету") | |
