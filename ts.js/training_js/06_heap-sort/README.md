# 🏔️ Сортировка кучей (Heap Sort)

## Что такое куча (Heap)?

Куча — это бинарное дерево с особым правилом:  
**каждый родитель ≥ своих детей** (Max-Heap)

Это означает: **корень дерева всегда хранит максимум**.

Хитрость: куча хранится в **обычном массиве**!

```
Массив: [10, 7, 6, 5, 3, 2, 1]
         0   1  2  3  4  5  6  ← индексы

Дерево:
           10        ← индекс 0
          /   \
         7     6     ← индексы 1, 2
        / \   / \
       5   3 2   1   ← индексы 3, 4, 5, 6

Для узла i:
  Левый ребёнок  → 2*i + 1
  Правый ребёнок → 2*i + 2
  Родитель       → Math.floor((i-1) / 2)
```

Проверяем: 10 ≥ 7 ✅, 10 ≥ 6 ✅, 7 ≥ 5 ✅, 7 ≥ 3 ✅ — это Max-Heap!

## Как работает алгоритм

Heap Sort работает в два этапа:

### Этап 1: Построить Max-Heap

```
[4, 10, 3, 5, 1]  →  buildHeap  →  [10, 5, 3, 4, 1]
                                         ↑ максимум в корне!
```

### Этап 2: Извлекать максимум и класть в конец

```
[10, 5, 3, 4, 1]
  ↑ максимум

Шаг 1: Меняем arr[0] ↔ arr[4]: [1, 5, 3, 4, | 10]
        Восстанавливаем кучу:   [5, 4, 3, 1, | 10]

Шаг 2: Меняем arr[0] ↔ arr[3]: [1, 4, 3, | 5, 10]
        Восстанавливаем кучу:   [4, 1, 3, | 5, 10]

Шаг 3: Меняем arr[0] ↔ arr[2]: [3, 1, | 4, 5, 10]
        Восстанавливаем кучу:   [3, 1, | 4, 5, 10]

Шаг 4: Меняем arr[0] ↔ arr[1]: [1, | 3, 4, 5, 10]

Результат: [1, 3, 4, 5, 10] ✅
```

## Функция heapify — восстановление кучи

```javascript
function heapify(arr, n, i) {
  let largest = i;
  const left  = 2 * i + 1;
  const right = 2 * i + 2;

  if (left < n  && arr[left]  > arr[largest]) largest = left;
  if (right < n && arr[right] > arr[largest]) largest = right;

  if (largest !== i) {
    [arr[i], arr[largest]] = [arr[largest], arr[i]];  // обмен
    heapify(arr, n, largest);   // "просеиваем" вниз
  }
}
```

## Сложность

| Случай     | Время      | Почему?                                           |
|------------|------------|---------------------------------------------------|
| Лучший     | O(n log n) | Построение кучи O(n), n извлечений × O(log n)    |
| Средний    | O(n log n) | То же самое                                       |
| Худший     | O(n log n) | **Всегда**, независимо от данных!                 |
| **Память** | **O(1)**   | Сортируем in-place, стек рекурсии O(log n)        |

---

## Сравнение с другими O(n log n) алгоритмами

| Алгоритм  | Время      | Память   | Стабильный | На практике |
|-----------|------------|----------|------------|-------------|
| Quicksort | O(n log n)*| O(log n) | ❌         | 🥇 Самый быстрый |
| Merge Sort| O(n log n) | O(n)     | ✅         | 🥈 Предсказуемый |
| Heap Sort | O(n log n) | O(1)     | ❌         | 🥉 Медленнее из-за кэша |

*Quicksort O(n²) в худшем случае, но это редко.

Heap Sort медленнее Quicksort на практике из-за **плохой cache-локальности**:  
доступ к элементам кучи "прыгает" по памяти (индексы 0, 1, 2, 3, 4 → потом 0, 2, 5, 11...).  
Процессор не может эффективно кэшировать такие обращения.

---

## Когда применять?

- Нужен **гарантированный O(n log n)** без O(n²) худшего случая
- Нужна **минимальная память** (O(1) — лучше Merge Sort)
- Embedded-системы с ограниченной памятью
- Priority Queue (куча сама по себе очень полезна!)
