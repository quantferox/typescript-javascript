# 🫧 Сортировка пузырьком (Bubble Sort)

## Аналогия из жизни

Представь, что ты стоишь в очереди из людей разного роста, и тебе нужно выстроить их от низкого к высокому.  
Ты идёшь вдоль очереди и смотришь на каждую пару соседей: если левый выше правого — просишь их поменяться.  
После первого прохода **самый высокий** окажется в конце.  
Повторяешь — второй по высоте встанет предпоследним.  
И так далее...

---

## Как работает алгоритм

```
Входной массив: [5, 3, 8, 4, 2]

Проход 1: сравниваем соседей, "всплывает" максимум → 8
  [5,3] → меняем → [3,5,8,4,2]
  [5,8] → оставляем
  [8,4] → меняем → [3,5,4,8,2]
  [8,2] → меняем → [3,5,4,2,8] ← 8 на месте ✅

Проход 2: работаем с [3,5,4,2] → всплывает 5
  → [3,4,2,5,8] ← 5 на месте ✅

Проход 3: работаем с [3,4,2] → всплывает 4
  → [3,2,4,5,8] ← 4 на месте ✅

Проход 4: работаем с [3,2] → всплывает 3
  → [2,3,4,5,8] ← готово ✅
```

## Код с объяснением

```javascript
function bubbleSortOptimized(arr) {
  const n = arr.length;

  for (let i = 0; i < n - 1; i++) {
    let swapped = false;           // был ли обмен на этом проходе?

    for (let j = 0; j < n - 1 - i; j++) {
      if (arr[j] > arr[j + 1]) {
        [arr[j], arr[j + 1]] = [arr[j + 1], arr[j]];  // обмен
        swapped = true;
      }
    }

    if (!swapped) break;           // массив уже отсортирован — стоп!
  }

  return arr;
}
```

**Почему `n - 1 - i` во внутреннем цикле?**  
После `i` итераций последние `i` элементов уже на своих местах.  
Нет смысла проверять их снова — экономим время.

---

## Сложность

| Случай        | Время  | Почему?                                          |
|---------------|--------|--------------------------------------------------|
| Лучший (✅)   | O(n)   | Массив уже отсортирован → 1 проход без обменов   |
| Средний       | O(n²)  | Вложенные циклы по n элементам                   |
| Худший (❌)   | O(n²)  | Обратный порядок → максимум обменов              |
| **Память**    | **O(1)**| Сортируем "на месте", 1 переменная для обмена   |

---

## Достоинства и недостатки

| ✅ Плюсы | ❌ Минусы |
|----------|-----------|
| Очень простой для понимания | Медленный на больших данных |
| Стабильный (порядок дубликатов сохраняется) | Квадратичная сложность O(n²) |
| In-place (O(1) памяти) | Не используется в production |
| Хорошо работает на почти готовых данных | |

---

## Когда применять?

- 📚 **Учёба** — лучший алгоритм для понимания сортировок
- 📐 **Маленькие массивы** (< 10 элементов) — разница незаметна
- 🔍 **Почти отсортированные данные** — с оптимизацией работает за O(n)

В реальных проектах предпочитают **Insertion Sort** (такой же простой, но быстрее), либо **Quicksort/Mergesort**.
