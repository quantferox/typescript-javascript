# ⚡ Быстрая сортировка (Quicksort)

## Аналогия из жизни

Ты сортируешь стопку бумаг по дате. Берёшь одну бумагу — "эталон" (pivot).  
Кладёшь в левую кучку всё, что **раньше эталона**, в правую — всё что **позже**.  
Теперь эталон уже стоит на **точно правильном месте**!  
Повторяешь то же самое для левой и правой кучки.

Стратегия называется **"разделяй и властвуй"** (Divide and Conquer).

---

## Как работает алгоритм

```
[3, 6, 8, 10, 1, 2, 1],  pivot = 3 (средний)

Разбиение:
  Меньше 3:  [1, 2, 1]
  Равно 3:   [3]
  Больше 3:  [6, 8, 10]

Рекурсия:
  [1, 2, 1] → pivot=2 → [1,1] | [2] | [] → [1, 1, 2]
  [6, 8, 10] → pivot=8 → [6] | [8] | [10] → [6, 8, 10]

Результат: [1, 1, 2, 3, 6, 8, 10] ✅
```

---

## Разбиение (Partition) — сердце алгоритма

```javascript
function partitionLomuto(arr, low, high) {
  const pivot = arr[high];  // pivot = последний элемент
  let i = low - 1;          // граница "зоны малых"

  for (let j = low; j < high; j++) {
    if (arr[j] <= pivot) {
      i++;
      [arr[i], arr[j]] = [arr[j], arr[i]];  // расширяем зону малых
    }
  }

  // Ставим pivot на своё место
  [arr[i + 1], arr[high]] = [arr[high], arr[i + 1]];
  return i + 1;  // индекс pivot — он уже на своём месте навсегда!
}
```

После partition pivot **больше никогда не перемещается** — он уже на финальной позиции.

## Визуализация разбиения Ломуто

```
arr = [3, 1, 4, 1, 5, 2],  pivot = 2

         i   j
[3, 1, 4, 1, 5, 2]
 ↑                        i = -1, j идёт вправо

j=0: arr[0]=3 > 2 → пропускаем
j=1: arr[1]=1 ≤ 2 → i=0, swap(0,1) → [1, 3, 4, 1, 5, 2]
j=2: arr[2]=4 > 2 → пропускаем
j=3: arr[3]=1 ≤ 2 → i=1, swap(1,3) → [1, 1, 4, 3, 5, 2]
j=4: arr[4]=5 > 2 → пропускаем

Финал: swap(i+1=2, high=5) → [1, 1, 2, 3, 5, 4]
                                     ↑ pivot 2 на позиции 2 ✅
```

## Сложность

| Случай     | Время     | Почему?                                               |
|------------|-----------|-------------------------------------------------------|
| Лучший     | O(n log n)| Pivot делит массив пополам каждый раз                |
| Средний    | O(n log n)| В среднем хорошее разбиение                          |
| Худший     | O(n²)     | Pivot всегда min или max → разбиение 1 и n-1         |
| **Память** | **O(log n)**| Стек рекурсии (глубина дерева)                     |

---

## Когда возникает худший случай O(n²)?

При выборе первого/последнего элемента как pivot **на уже отсортированных данных**:

```
[1, 2, 3, 4, 5],  pivot = последний = 5
→ левая часть: [1,2,3,4], правая: []   (разбиение 4+0, а не 2+2)
→ следующий pivot = 4
→ [1,2,3], []
→ ...это O(n²)!
```

**Решение:** случайный pivot или медиана трёх.
